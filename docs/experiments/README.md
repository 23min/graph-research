Each experiment lives in its own directory keyed by ID.

## Directory template

```
EXP-NN-slug/
├── hypothesis.md   — claim being tested, why it matters, what would falsify it
├── setup.md        — corpora, baselines, metrics, budget, seed, code SHA, config paths
├── results/        — raw outputs (CSV/JSON), regenerated by scripts
├── discussion.md   — what the numbers say and do not say; caveats; follow-up
└── design-notes/   — (optional) exploratory design material supporting the experiment
```

## Conventions

- IDs are sequential and stable. Do not reuse.
- Reference the experiment ID in commit messages (`feat(EXP-03): ...`).
- Negative results are first-class and follow the same template.
