# Research Discipline Rules

This repo is a research project on automated configuration and multi-objective optimization of layered and metro-map graph layout algorithms. The rules below keep the work reproducible, citable, and defensible under scrutiny.

## 1. Experiments are first-class artefacts

- Every experiment has an `EXP-NN-slug` ID, created at the start and stable forever.
- Each experiment lives in `docs/experiments/EXP-NN-slug/` with at minimum:
  - `hypothesis.md` — what claim the experiment tests, why it matters, what would falsify it
  - `setup.md` — corpora, baselines, metrics, budget, seed, code SHA, configuration files
  - `results/` — raw outputs (CSVs, JSONs) regenerated by scripts, never handwritten
  - `discussion.md` — what the numbers say, what they do not say, caveats, follow-up
- Negative results are written up with the same structure. The v1 weighted-sum cycling story is `EXP-01-weighted-sum-cycling`.
- Every commit message that advances an experiment references its ID (`feat(EXP-03): wire NSGA-II archive`).

## 2. Bibliography is the single source of truth for citations

- Every paper referenced anywhere in the repo has an entry in `docs/literature/bibliography.bib`.
- Entries include `title`, `author`, `year`, `venue` (journal/conference), `doi` (preferred) or `url` (fallback).
- Per-paper reading notes live at `docs/literature/notes/<bibkey>.md`, keyed to the BibTeX key.
- Thematic reviews across multiple papers live at `docs/literature/reviews/<topic>.md`.
- PDFs live in `docs/literature/pdfs/` which is a symlink to a Dropbox bind-mount (`/workspaces/research-pdfs`); gitignored either way.
- If a paper is cited in a methodology doc or experiment, its BibTeX key must already exist — no TODO-citations.

## 3. Decisions are numbered ADRs

- Architectural and methodological decisions are recorded in `docs/decisions/NNNN-slug.md`.
- Use sequential 4-digit numbering (`0001-four-corpora.md`).
- ADRs capture: context, options considered, decision, consequences. Short is fine; the point is durability.
- Superseded ADRs are not deleted — they are marked `status: superseded` with a link forward.

## 4. Reproducibility protocol

Every reported result — in an experiment, a methodology doc, a write-up — carries:
- RNG **seed** (committed integer)
- **Configuration file** path (committed JSON/YAML)
- **Code SHA** of the repo at run time (and the `dag-map` submodule SHA)
- **Split version hash** (the `splits.json` `version` field and its SHA-256)

Results that cannot produce all four are not reported. Sketch / exploratory runs get prefixed with `scratch-` and live in a gitignored directory.

## 5. Results are regenerated, never handwritten

- Numeric outputs in `docs/results/` and `docs/experiments/<EXP>/results/` are produced by scripts committed under `scripts/` or `bench/`.
- File headers include the script path that produced them (e.g. `# generated by scripts/run-EXP-03.mjs at <SHA>`).
- When inputs change, the script is re-run and the result file is regenerated. No manual edits.

## 6. PDF handling

- Primary: Dropbox bind-mount at `~/Dropbox/graph-research-pdfs/` → `/workspaces/research-pdfs` → symlinked at `docs/literature/pdfs/`.
- Fallback (CI, collaborators without Dropbox): `node scripts/fetch-pdfs.mjs` pulls open-access PDFs resolved from `bibliography.bib`. Paywalled papers fail gracefully and are listed for manual acquisition.
- Never commit PDFs.

## 7. Held-out discipline

- The corpora split (`bench/fixtures/splits.json`) is frozen by content hash. Training-side code must route fixture reads through `bench/fixtures/split-loader.mjs`.
- The CI guard (`bench/fixtures/__tests__/split-guard.test.mjs`) fails the build on direct fixture reads from training code paths.
- Regenerating the split requires an ADR entry and a version bump.

## 8. No unauthorized commits or pushes

Inherited from the framework, re-stated for emphasis: never commit or push without explicit human approval. "Continue" and "ok" do not count. This rule applies equally to research artefact commits.
